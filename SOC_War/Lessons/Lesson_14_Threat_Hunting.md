# Lesson 14 — Threat Hunting & Proactive Detection

## Core Idea
Incident Response = reactive (alert fires, you respond).
Threat Hunting = proactive (you search for hidden threats that haven't triggered an alert).

One-line definition:
"I suspect attackers may behave in X way. Let me search the telemetry for proof."

---

## The Hunt Flow (telemetry-corrected)

Attacker behavior
  -> Telemetry generated
  -> Logs contain that telemetry
  -> Query searches the telemetry

Experienced analysts ask "What telemetry do we have?" BEFORE "What query do I write?"

Correct learning order:
Attacker behavior -> what evidence appears -> what logs hold it -> what fields matter -> THEN write query. Never syntax-first.

---

## Telemetry — The Missing Bridge

Definition: Data generated by systems, apps, users, and networks that describes
what is happening inside an environment. SOC analysts never see the attacker
directly; they see telemetry and reconstruct the story.

Telemetry types:
- Process telemetry  -> what ran, who launched it, parent->child relationships
- Network telemetry  -> src_ip, dest_ip, dest_port, bytes, protocol
- Auth telemetry     -> logins, failures, successes, SOURCE of login
- File telemetry     -> files created, modified, deleted

Process trees = stories:
powershell.exe -> certutil.exe -> payload.exe
is not three random programs; it is one attack story.
Key field = ParentImage (shows who launched what).

Sysmon (one line):
Windows Event Logs say something happened; Sysmon says HOW it happened,
who started it, what it connected to, and what it spawned.
= telemetry amplifier for Windows.

Common Sysmon EventCodes:
- EventCode=1  -> Process creation
- EventCode=3  -> Network connection
- EventCode=7  -> Image/DLL loaded
- EventCode=11 -> File created
- EventCode=12 -> Registry modified

---

## Key Terms Grounded

SMB = Server Message Block. Windows file/printer sharing protocol. Port 445.

Admin shares (hidden, auto-created on every Windows machine):
- C$     -> entire C: drive
- ADMIN$ -> Windows system directory
- IPC$   -> inter-process communication
($ = hidden.) An attacker with admin creds can reach \\TARGET\C$ = whole drive,
no malware install needed.

Lateral movement = moving sideways from one compromised host to other hosts.
PSEXESVC.exe spawning = classic lateral movement IOC (PsExec).

---

## The Hunt Table

| Behavior | Telemetry type | Key fields |
|----------|----------------|------------|
| Brute force: many failed logins then success, same IP/user | Auth | failed count, success count, src_ip, user, time window |
| LOLBin: PowerShell spawns process chain to run malware | Process | Image, ParentImage, CommandLine (-enc/IEX/Invoke-), PID, User |
| LOLBin: that process downloads payload from external IP | Network | src_ip, dest_ip, dest_port, bytes, protocol |
| Lateral movement: host connects to another over SMB | Network | src_ip, dest_ip, dest_port=445 |
| Lateral movement: admin creds used on target | Auth | successful login, SOURCE = unexpected host, time |
| Lateral movement: remote command runs on target | Process | PSEXESVC.exe, ParentImage, CommandLine, User |

Once the table is filled, the SPL query is just translation.

## Worked Hunts

### Hunt 1 — Brute Force Success
Behavior: many password attempts, eventual success, same IP, same user.
Telemetry: Auth.
Key fields: failed count, success count, src_ip, user, time window.
Logic: failures > 5 AND success > 0 from same source = brute force success.

### Hunt 2 — LOLBins (PowerShell abuse)
Behavior: attacker uses legitimate powershell.exe to download/run malware stealthily.
Telemetry: BOTH process and network.
- Process: Image, ParentImage, CommandLine (look for -enc / IEX / Invoke-), PID, User
- Network: src_ip, dest_ip, dest_port, bytes
Insight: process shows the command; network shows the connection it created.
Correlate both = full story.

### Hunt 3 — Lateral Movement via SMB
Behavior: compromised host uses stolen admin creds to reach other hosts over SMB.
Telemetry: network + auth + process.
- Network: src_ip, dest_ip, dest_port=445
- Auth: successful login on target FROM an unexpected source host
- Process (on target): PSEXESVC.exe -> cmd.exe/powershell.exe
Insight: no single signal proves it; all three (same source, same timeframe,
same target) = lateral movement, high confidence.

---

## Key Learnings (Lesson 14)
1. Telemetry is the bridge between attacker behavior and the query.
2. Read SPL as a question to logs, not as code.
3. Process trees are stories; ParentImage is the thread.
4. Most real hunts are multi-telemetry (correlation beats single signals).
5. Fill the hunt table first; the query is just translation of that table.
